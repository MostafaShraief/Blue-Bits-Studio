## STRICT GENERATION RULES

### 1. Question Count (ANTI-HALLUCINATION)

* First, analyze the **current lecture only** and identify how many **distinct, clearly explained concepts** it contains.
* Generate **only as many questions as the current lecture can fully support**, up to a **maximum of 25 questions**.
* If the lecture supports fewer than 25 questions, generate fewer questions.
* **DO NOT invent, extend, infer, or assume information not explicitly stated in the current lecture.**
* If a concept is not clearly covered in the current lecture, it must NOT be used.

---

### 2. Correct Answer Distribution (ANTI-BIAS)

* Across all generated questions in this output:

  * The indices `[0, 1, 2, 3]` must be used **as evenly as possible**.
  * No single index may appear in more than **40%** of the questions.
* Before finalizing the output, verify and rebalance the correct answer positions if needed.

---

### 3. Options Construction

* Exactly **four options** per question.
* Remove all labels such as A, B, ج, etc.
* Options must be:

  * Linguistically balanced
  * Plausible and technically reasonable
* Avoid:

  * Making the correct answer the longest option
  * Translating only the correct option
  * Repeating structural or stylistic patterns across options

---

### 4. `type` Field

* Use `"quiz"` when there is **exactly one** correct answer.
* Use `"regular"` only when **multiple answers are explicitly correct** according to the current lecture.
* If uncertain, default to `"quiz"`.

---

### 5. Explanation Field (ANTI-CITATION / ANTI-JSON-BREAK)

* Explanations must:

  * Be short and factual, less than 200 chars
  * Be directly grounded in the **current lecture wording**
* STRICTLY FORBIDDEN in explanations:

  * Citations of any kind
  * Placeholders or reference tokens
  * Square brackets `[` or `]`
  * External knowledge
* If a valid explanation cannot be written using only the current lecture, **do not generate the question**.
